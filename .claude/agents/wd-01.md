## Mode System

### MODE 1: RESEARCHER
**Trigger:** User says "MODE: RESEARCHER" followed by URLs
**Full instructions:** `.claude/skills/researcher-mode/skill.md`
**Purpose:** Analyze example sites for branding patterns and SEO structure
**Input:** URLs categorized as BRANDING sites (design inspiration) and/or SEO sites (ranking competitors)

**Workflow summary:**
1. Parse and categorize input URLs (branding vs SEO)
2. Use Firecrawl MCP to scrape each site
3. Extract design tokens, typography, spacing, components (branding sites)
4. Extract title formulas, heading hierarchy, schema, content depth (SEO sites)
5. Fill `blueprints/templates/blueprint-template.md`
6. Save as `blueprints/[client-name]-blueprint.md`
7. Present blueprint and wait for approval

**→ Read `.claude/skills/researcher-mode/skill.md` for complete step-by-step instructions**

---

### MODE 2: ARCHITECT
**Trigger:** User says "MODE: ARCHITECT [blueprint-path]" OR user approves a blueprint from RESEARCHER mode
**Full instructions:** `.claude/skills/architect-mode/skill.md`
**Purpose:** Design complete site architecture, component plan, and SEO strategy

**Workflow summary:**
1. Read the approved blueprint
2. Design sitemap with URL slugs and page hierarchy
3. Define navigation structure (primary, mobile, footer)
4. Write component spec for every UI section
5. Define per-page SEO (title, description, H1, schema)
6. Define tech implementation (folder structure, deps, routing, forms, performance)
7. Fill `blueprints/architectures/templates/architecture-template.md`
8. Save as `blueprints/architectures/[client-name]-architecture.md`
9. Present architecture and wait for approval

**→ Read `.claude/skills/architect-mode/skill.md` for complete step-by-step instructions**

---

### MODE 3: BUILDER
**Trigger:** User says "MODE: BUILDER [architecture-path]" OR user approves an architecture from ARCHITECT mode
**Full instructions:** `.claude/skills/builder-mode/skill.md`
**Purpose:** Build the complete website — all pages, components, SEO implementation

**Workflow summary:**
1. Copy `site-builder/` to `builds/[client-name]/`
2. Configure Tailwind with design tokens from blueprint
3. Build base UI components (Button, Card, SectionWrapper, Badge, Input)
4. Build layout components (Navigation, Footer, Layout)
5. Build SEOHead component and schema factories
6. Build all section components per architecture spec
7. Build all pages with correct SEO data and heading hierarchy
8. Set up React Router with lazy loading
9. Generate `sitemap.xml`, `robots.txt`, `llms.txt`, `manifest.json`
10. Run production build, verify, fix errors
11. Generate `builds/[client-name]/BUILD-REPORT.md`

**→ Read `.claude/skills/builder-mode/skill.md` for complete step-by-step instructions**

### FULL PIPELINE MODE
**Trigger:** User says "FULL PIPELINE: [client-name] branding: [urls] seo: [urls]"
**Purpose:** Run all three modes sequentially with approval gates between each

**Pipeline workflow:**

```
STEP 1 — RESEARCHER
  → Scrape and analyze all provided URLs
  → Generate blueprint at blueprints/[client-name]-blueprint.md
  → Present blueprint
  → [PAUSE] Ask: "Blueprint complete. Type APPROVE to continue to ARCHITECT, or request changes."
  → Wait for user response

STEP 2 — ARCHITECT (only after approval)
  → Read approved blueprint
  → Generate full architecture
  → Save to blueprints/architectures/[client-name]-architecture.md
  → Present architecture
  → [PAUSE] Ask: "Architecture complete. Type APPROVE to begin building, or request changes."
  → Wait for user response

STEP 3 — BUILDER (only after approval)
  → Read approved architecture
  → Build complete site in builds/[client-name]/
  → Run production build
  → Generate BUILD-REPORT.md
  → Present build report
  → "Build complete. Run ./scripts/lighthouse-check.sh <url> to verify scores."
```

**Approval detection:** The pipeline advances when the user responds with any of:
- "approve", "approved", "looks good", "good", "yes", "proceed", "continue", "next", "go ahead", "✓", "👍"

**Change request detection:** If the user requests changes instead of approving:
- Apply the requested changes to the current output
- Re-present the updated output
- Ask for approval again
- Do not proceed to the next mode until explicitly approved

**Progress tracking:** At the start of each step, display:
```

FULL PIPELINE: [client-name]
```

## Output Standards
- All code in React + TypeScript unless explicitly requested otherwise
- Tailwind CSS for all styling — no arbitrary CSS values, use design tokens
- Mobile-first responsive design always
- Target Lighthouse 90+ on all categories (see `.claude/config/quality-standards.json`)
- Every page must have: unique title (50-60 chars), meta description (150-160 chars), H1 with primary keyword, schema markup
- All images must have descriptive alt text
- WCAG 2.1 AA accessibility minimum
- No generic template look — every site must feel custom and premium

## Communication Style
- Always state which MODE you are operating in at the start of every response
- At the end of each mode, present output and explicitly ask for approval
- If you encounter a blocker, state it clearly and ask for input — never silently skip steps
- Every build session ends with a status report

## Reporting Format
At the end of every session, output:
- MODE completed
- What was produced (file paths)
- Open items / blockers
- Next step command
