This project is the WD-01 Web Developer Agent system. It builds professional, SEO-optimized websites through a three-mode pipeline: RESEARCHER → ARCHITECT → BUILDER.

## Mode Activation

When the user says **"MODE: RESEARCHER"**, **"MODE: ARCHITECT"**, or **"MODE: BUILDER"**:
1. Announce the active mode
2. Read the corresponding skill file listed above
3. Follow its instructions step-by-step without skipping phases

When the user says **"FULL PIPELINE"**, follow the complete sequential workflow:
1. **RESEARCHER** → produce blueprint → present → ask for approval → pause
2. **ARCHITECT** → produce architecture → present → ask for approval → pause
3. **BUILDER** → build the site → produce build report → present

Never advance to the next mode without explicit user approval.

## Quality Standards

- Read `.claude/config/quality-standards.json` for minimum score thresholds on every build
- Read `qa/checklists/qa-checklist.md` before declaring any build complete
- Every site must pass all 23 QA checklist items before delivery

Key minimums:
- Lighthouse Performance ≥ 90
- Lighthouse SEO ≥ 95
- Lighthouse Accessibility ≥ 90
- LCP < 2.5s | CLS < 0.1 | INP < 200ms | FCP < 1.8s
- WCAG 2.1 AA
- Title tags: 50–60 chars | Meta descriptions: 150–160 chars

## MCP Servers

Both MCP servers are configured in `.claude/settings.json`:

- `firecrawl` — web scraping (use in RESEARCHER mode to analyze competitor sites)
