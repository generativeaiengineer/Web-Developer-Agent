## 5. RECOMMENDATIONS & NOTES

### Competitive Differentiation Opportunities

1. **Speed & Transparency** — Both Hays and Michael Page have enterprise-bloated sites with slow load times. SkaloRecruit should be noticeably faster (sub-2s LCP). This is a real differentiator.

2. **Modern Visual Identity** — The established players all use legacy color schemes (navy + stock grey). The orange accent (#FF6B35) gives SkaloRecruit immediate visual differentiation in a crowded SERP or ad impression.

3. **SEO Gap — Short Titles** — Michael Page consistently uses titles under 40 characters (e.g., "Career Advice | Michael Page" = 30 chars). This leaves keyword real estate on the table. SkaloRecruit should use full 50–60 char titles on every page.

4. **GEO / AI Search** — Neither Hays nor Randstad have a visible `llms.txt` file. Including one from day one positions SkaloRecruit ahead in AI-search discovery (ChatGPT, Perplexity citations).

5. **Schema Coverage** — Randstad implements solid schema (Organization, WebSite, WebPage, BreadcrumbList). SkaloRecruit should match this baseline and add FAQPage and Service schema on all service/industry pages — this is above what competitors currently implement.

6. **Candidate vs. Employer Split** — Hays [EXTRACTED] uses clear dual CTAs: "Find your next job now" vs "Hire top talent today". SkaloRecruit should replicate this split navigation model from the very first viewport — it reduces bounce from wrong-audience visits.

7. **Content Depth on Service Pages** — Randstad service pages are 800–1,000 words [EXTRACTED]. Aiming for 1,500–2,000 words on SkaloRecruit service pages with FAQ sections will provide a content depth advantage for ranking.

8. **Stats/Social Proof** — Hays prominently uses: 21 specialisms, 30 countries, 57 years, 10,000+ employees [EXTRACTED]. SkaloRecruit should define 4 compelling stats at launch (e.g., placements made, average days to hire, client retention rate, industries served).

### Technical Warnings

- **Anti-scraping**: All four reference sites use aggressive scraping protection (robertwalters.com = 403, michaelpage.com = partial Cloudflare block). SkaloRecruit should implement respectful bot access — allow `GPTBot`, `ClaudeBot`, `PerplexityBot` in `robots.txt` while blocking scrapers.
- **robots.txt model**: Michael Page [EXTRACTED] correctly blocks `/job-apply/`, admin paths, and salary filter parameter pages — adopt this pattern.
- **Job listing pages**: If dynamic job listings are added in Phase 2, use `?` parameter pages with canonical pointing to the clean URL, or noindex filtered pages.
